Game Engine Concepts

[ architecture ]

Scene Graph

Spatial

The base entity for scene graph nodes. It maintains a link to a parent, it's local transforms and the world's transforms

Coordinate System

Node

An invisible "handle" for scene objects which groups Geometries and other Nodes (audio, light, etc) together

Geometry

Leaf node in the scene graph. A visible object on the screen which stores the object's looks (mesh and material)

Mesh & Material

  • Mesh - made up by polygons
  • Material
    • "Unshaded" materials - look somewhat abstract as they ignore lighting and shading
    • Illuminated materials require a light source added to at least one of their parent nodes

Camera

Bounding Volume

3D objects can have irregular forms, other that basic 3D geometrical shapes. To aid mathematical computation, they are wrapped in a bounding volume which pragmatically represents the object as a sphere, cube, cylinder, etc.

Ray Casting

A vector starting from the certain point of an entity having either a finite or infinite length. It determines which objects in the 3D space it intersected

(useful to compute distance)

How would you use 3D simulation?

  • Ubicomp
  • Context-Awareness & Context
  • Context-Aware Systems
  • Ubicomp = viziunea acestui concept e sa integreze cat mai bine tehnologia in vietile noastre de zi cu zi. Sistemele sunt menite sa preia din interactiunea explicita si sa o faca cat mai implicita bazat pe starea curenta a sistemului
  • Context-Awareness = parte a ubicomp care se ocupa de a lega schimbarile in environment de sisteme care, de altfel, sunt static
  • Context = orice informatie care poate caracteriza starea unei entitati, unde o entitate poate fi o persoana, un loc, un obiec fizic sau device
  • Context-Aware System = folosesc date de context pt a oferi utilizatorului servicii bazate pe starea sa curenta. E.g. -> the "city guide" example
